# Лабораторная работа №1
## «Оцифровка зданий в OpenStreetMap с использованием веб‑редактора»

### 1. Цель работы
Освоить практические навыки работы с OpenStreetMap (OSM) в веб‑редакторе iD: поиск недостаточно оцифрованных территорий, добавление зданий, заполнение семантики, публикация изменений, экспорт данных и подготовка pull‑request.

---

## 2. Содержание задания
1. Найти участок карты в OSM, где отсутствует значительная часть зданий.
2. Выполнить оцифровку недостающих зданий.
3. Заполнить семантические атрибуты для каждого здания.
4. Дождаться публикации изменений в основной базе OSM.
5. Скачать обновлённые данные.
6. Преобразовать их в GeoJSON.
7. Подготовить pull‑request в репозиторий, приложив полученный файл.

---

## 3. Порядок выполнения работы

### 3.1. Выбор территории
1. Откройте сайт **https://www.openstreetmap.org**.
2. Выполните регистрацию.
3. Найдите район, где здания нанесены частично или отсутствуют. Примеры территорий, на которых возможна оцифровка:
https://www.openstreetmap.org/#map=16/53.59540/49.28948
https://www.openstreetmap.org/#map=19/13.768197/109.219482
4. Зафиксируйте выбранную область.

---

### 3.2. Оцифровка объектов
1. Нажмите кнопку **Правка** — откроется веб‑редактор **iD**.
2. Выберите инструмент **Полигон** для здания, **Линия** для дорог, **Точка** для POI (point of interest).
3. Обведите объект по спутниковому снимку. При необходимости переключайте подложки:
<img width="300" height="500" alt="image" src="https://github.com/user-attachments/assets/715e9fe5-feac-406c-a1ea-06d4745f1f72" />

4. Для ортогонализации полигонов можно использовать кнопку `Q` на клавиатуре.
5. Укажите тип объекта в соответствии с классификатором OSM.

Объектами могут быть здания, дороги, POI. Цель - выполнить оцифровку как минимум **10 объектов**. Также возможно уточнение уже внесенных объектов, если их графика или семантика не соответствует текущему состоянию.

Дополнительно при оцифровке и при заполнении семантики можно использовать панорамы и другие картографические сервисы:
https://www.google.com/maps
https://maps.yandex.ru/

---

### 3.3. Заполнение семантики.

Обязательные для заполнения поля для зданий:
- число этажей
- адресная информация

Обязательные для заполнения поля для дорог:
- дорожное покрытие

Обязательные для заполнения поля для точечных объектов:
- название
- адресная информация

---

### 3.4. Публикация изменений
1. Нажмите **Сохранить**.
2. В комментарии к изменениям укажите что было сделано (например, «Добавлены недостающие места»).
3. Устраните ошибки верификации пакета правок, если они обнаружены
3. Подтвердите загрузку.
4. Дождитесь обновления данных на общей карте.

---

### 3.5. Загрузка обновлённых данных
После того как изменения стали видны на карте:

1. Перейдите на **https://www.openstreetmap.org**.
2. Спозиционируйте на место, которые вы редактировали.
3. Нажмите на кнопку "Экспорт" и выполните экспорт данных в формат osm.
4. Выполните преобразование из формата osm в GeoJSON, например, с использованием https://tyrasd.github.io/osmtogeojson/

---

### 3.6. Подготовка pull‑request
1. Сделайте форк данного репозитория
2. Откройте новый PR в данный репозиторий. В PR приложите:
- ссылку на changeset в OSM
- полученный GeoJSON-файл

3. Исправите замечания после review, если они будут.
